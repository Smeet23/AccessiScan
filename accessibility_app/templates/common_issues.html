Fix Common Accessibility Issues

Accessibility ensures your website can be used by everyone, including people with disabilities. Below are some of the most common issues and actionable tips to fix them.

1. Missing Alt Text

All images must include descriptive <img alt="..."> attributes. This helps screen readers describe the image to users who can't see them.

Example: Use <img src="logo.png" alt="Company Logo"> instead of leaving alt empty.

2. Poor Color Contrast

Text must have enough contrast against its background to be readable by people with visual impairments.

Check contrast with WebAIM and aim for at least a 4.5:1 ratio for normal text (WCAG AA).

3. Missing Form Labels

Every input field must be associated with a <label> for clarity and screen reader support.

Good: <label for="email">Email:</label> <input id="email">

4. Lack of Keyboard Navigation

All interactive elements should be accessible via keyboard (Tab, Enter, Space). Avoid relying solely on mouse or touch input.

Tip: Use tabindex="0" for custom elements and avoid using tabindex="-1" unless necessary.

5. Missing Page Landmarks

Use semantic HTML tags or ARIA roles to define page structure. This helps assistive tools understand the layout.

Use: <nav>, <header>, <main>, <footer>.
